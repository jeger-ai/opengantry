## Problem

The specimen `MANIFEST.json` serves adopters (`gapman init` template) and dogfood simultaneously. `ui` / `logic` skills point at `src/components/`, `src/lib/` (non-existent in opengantry repo); real dogfood uses `gapman` → `src/cli/`. `loadManifest()` reads only `REL_MANIFEST` — no merge/overlay. Phantom skills pollute triage, MSN-enforced prefixes (`gxt-manifest-lib.mjs`), and onboarding.

## Acceptance

- [ ] Optional overlay e.g. `.gitagent/foreman/MANIFEST.dogfood.json` (or env `GAPMAN_MANIFEST_OVERLAY`) merged in `loadManifest()` for opengantry only
- [ ] Overlay adjusts skills/path_risks for specimen (e.g. mark `ui`/`logic` inactive or remap defaults); template repos unchanged
- [ ] `gxt-manifest-lib.mjs` uses same merge rules for MSN prefixes
- [ ] `gapman doctor` warns when overlay missing in known specimen repo (or detect via marker file)
- [ ] `docs/DEVELOPMENT.md` + onboarding reference overlay; triage in specimen defaults to `gapman` skill hints
- [ ] Decision documented vs #13 (overlay vs split manifest)
