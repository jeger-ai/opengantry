## Problem

Adopters can point architecture at Confluence/wiki URLs via `kind: external` in `.gitagent/ARCHITECTURE.pointer.json`, and store credentials via `gapman arch cred set`, but gapman never **fetches** architecture content. `architectureRequiresDiscovery()` returns false for `external` without reachability check. Agents depend on ad-hoc MCP or manual copy-paste, breaking deterministic onboarding when external docs are the source of truth.

## Acceptance

- [ ] `gapman arch fetch` (or `arch pointer fetch`) resolves `kind=external`, applies creds from slot, HTTP GET with timeouts/size limits, optional SSRF guard (allowlist hosts from pointer)
- [ ] Cached snapshot under `.gitagent/history/architecture/` (git-ignored) with metadata (etag, fetched_at, content-type)
- [ ] Doctor: warn/fail if `access.required` and fetch fails or cache stale beyond configurable TTL
- [ ] MCP tool optional: `gxt_arch_fetch` returning summary + cache path (no secrets in response)
- [ ] Tests with mocked HTTP; fixture for bearer auth
- [ ] `ARCHITECTURE-ACCESS.md` updated: step uses CLI fetch before agent reads

## Priority

**Minor** — adoption enabler for enterprise wikis; credential path already shipped. Requires ADR + Teacher mission.
