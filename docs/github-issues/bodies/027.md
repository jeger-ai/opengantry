## Problem

Changed-mission verify logic lives in bash (`scripts/verify-pr-missions.sh`), duplicated conceptually with `.githooks/pre-push`. Contributors must know shell scripts and env vars (`BASE_SHA`, `HEAD_SHA`); no first-class CLI for local "verify what this branch touched." `program-workflow.ts` verify command has no `--changed-missions`, `--base`, or `--head`.

## Acceptance

- [ ] `gapman verify --changed-missions [--base <ref>] [--head <ref>]` mirrors `verify-pr-missions.sh` behavior (contamination, MSN-enforced paths, per-mission verify)
- [ ] Shared implementation in `src/cli/lib/` (single source; shell script becomes thin wrapper or deprecated)
- [ ] `--pre-push` semantics preserved for legislative stubs on changed missions only
- [ ] `--json` aggregate result (missions verified, failures, contamination)
- [ ] Tests port critical cases from `verify-pr-missions.test.ts`
- [ ] README + `docs/DEVELOPMENT.md` document CLI as canonical local command
