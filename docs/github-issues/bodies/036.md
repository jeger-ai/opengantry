## Problem

Enterprise CI and security dashboards expect **SARIF** (Code Scanning) or **JUnit** (test report UI) for gate failures. OpenGantry verify JSON (`VerifyResultPayload` from MSN-0028) is agent-oriented but not ingestible by standard platform widgets without custom adapters. No SARIF or JUnit output exists in the codebase.

## Acceptance

- [ ] `gapman verify --sarif-out <path>` and/or `--junit-out <path>` (mutually exclusive with human output; composable with `--json` policy documented)
- [ ] SARIF 2.1.0 document: one run per verify invocation; rules for git-proof, gate, trace, stale evidence with `GXT_*` rule ids; locations point to mission file + WORKER_LOG line when known
- [ ] JUnit: testcase per trace row / phase; failures include `error_code` and truncated `fix_hints`
- [ ] CI example snippet in `docs/ADOPTION.md`; optional upload step in template workflow (commented)
- [ ] Tests: golden SARIF/JUnit files for pass, git-proof fail, trace fail fixtures
- [ ] Implement on unified `runVerifyCore` (#44) — no duplicate verify orchestration
