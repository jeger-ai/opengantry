## Problem

Forbidden zones are enforced late (runtime exec in `forbidden-scan.ts`) or not at all (IDE edits). Missions can be legislated for work that **necessarily violates** manifest forbidden zones — e.g. intent referencing `.gitagent/foreman/` with skill `gapman`. `runLegislate` validates MSN/skill_key but performs **no FZ/intent path analysis**. Triage escalates on `path_risks` and `risk_keywords`, not forbidden-zone overlap. Violates RULES §4 intent.

## Acceptance

- [ ] Shared lib helper: given `skill_key` + intent (+ optional gate_command), detect manifest forbidden path overlap (reuse path normalization from `forbidden-scan.ts`)
- [ ] `gapman legislate` and MCP `gxt_draft_legislation` / `gxt_execute_legislation` reject with stable error (e.g. `FORBIDDEN_ZONE_IN_INTENT`) unless Teacher override flag documented in ADR (default: no override)
- [ ] Triage optionally surfaces same signal in `LEGISLATIVE_ESCALATION` reason (align with #28 explainability)
- [ ] Tests: intent mentioning `.gitagent/foreman/` + skill `gapman` → legislate fails
- [ ] Docs: MISSION-ARCHITECT interview step calls out FZ preflight

## Priority

**Major** — low-cost, high-signal governance; complements #14. Requires ADR + Teacher mission.
