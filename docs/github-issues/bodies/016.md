## Problem

Deterministic gates catch test/gate/trace failures but not **semantic architecture drift** (wrong layer, forbidden coupling, scope creep). No `ARCHITECTURE_RUBRIC.md` or async judge pipeline exists. `gapman verify` gate + trace is execution evidence, not architecture judgment. Teams need an optional **async LLM-as-judge** pass driven by a stable rubric, producing advisory findings tied to mission/MSN — without replacing deterministic verify or inventing architecture when pointer is `unset`.

## Acceptance

- [ ] ADR defines judge role vs Worker/Verifier/Teacher (SOD, Tier-2 advisory default)
- [ ] Shipped `ARCHITECTURE_RUBRIC.md` template (scoring dimensions, pass/fail/advisory thresholds, required inputs)
- [ ] CLI entry point (e.g. `gapman arch review --mission …`) that reads pointer + TARGET_ARCHITECTURE (#15) + diff scope and emits structured JSON + human summary
- [ ] Output recorded in git-ignored history or WORKER_LOG section with trace-friendly quote format; never blocks merge alone unless mission elevates to Tier-3
- [ ] Hard rule: pointer `kind=unset` or stub → judge refuses to score, points to ARCHITECTURE-DISCOVERY
- [ ] Tests with fixture diffs; documentation in `docs/ADOPTION.md` § honest limits

## Priority

**Minor** — valuable for regulated adopters; depends on #15 and stable architecture pointer workflow (#34 for external docs). Requires ADR + Teacher mission.
