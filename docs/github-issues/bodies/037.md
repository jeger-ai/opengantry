## Problem

Teacher stamps rely on **Git author email** matching `TEACHER.allowlist` — sufficient for process control but weak against credential compromise or misconfigured identity. `git-proof.ts` checks email only; no GPG signature verification or sigstore/cosign binding for `[MSN-*]` commits. `docs/COMPLIANCE-ISO.md` states trust model is Git identity + conventions, not signed artifacts.

## Acceptance

- [ ] ADR chooses mode(s): GPG signed commits, sigstore/cosign on `[MSN-*]` commits, or optional hardening layer (default off for backward compatibility)
- [ ] Config surface: e.g. `.gitagent/foreman/TEACHER.signing.json` or manifest extension with required key ids / sigstore issuer
- [ ] `gapman verify` git-proof phase: when enabled, stamp commit must carry valid signature matching allowlisted identity map
- [ ] Doctor: warn when signing required but `git log --show-signature` fails on latest Teacher stamp
- [ ] Document migration from email-only; CI fetch signatures where needed
- [ ] Tests with fixture repos containing signed/unsigned commits (GPG test key in fixtures)

## Priority

**Minor** — important for ISO/fintech narrative; higher effort; keep email-only path as default v1.x behavior. Requires ADR + Teacher mission.
