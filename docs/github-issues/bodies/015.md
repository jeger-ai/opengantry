## Problem

Architecture guidance is split between pointer JSON (`.gitagent/ARCHITECTURE.pointer.json`), markdown prose (`docs/ARCHITECTURE.md`), and gapman-specific CI scripts (`scripts/check-import-layers.mjs`). Agents and gates cannot **deterministically** evaluate "edit allowed here?" or "import violates layer rule?" from one versioned artifact. No `TARGET_ARCHITECTURE.yaml` exists in repo or templates; `gapman mission validate` validates missions, not architecture layers.

## Acceptance

- [ ] ADR + schema for `TARGET_ARCHITECTURE.yaml` (version field, layers, import rules, optional per-skill overlays)
- [ ] Init/upgrade scaffolds template; pointer or docs reference it explicitly
- [ ] `gapman doctor` validates file presence, schema, and stub/incomplete states (warn/fail policy documented)
- [ ] At least one deterministic check wired to the file (e.g. import-layer validation reads YAML rules, or legislate rejects missions whose `skill_key` conflicts with declared layers)
- [ ] Specimen repo (`docs/ARCHITECTURE.md` + new YAML) dogfoods the format
- [ ] Tests for valid/invalid fixtures; docs in `.gitagent/teacher/ARCHITECTURE-DISCOVERY.md` cross-linked

## Priority

**Major** — foundation for #16 and stronger legislate/triage. Requires ADR + Teacher mission before implementation.
