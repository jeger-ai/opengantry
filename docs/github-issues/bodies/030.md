## Problem

Only Cursor gets automatic mission/TMVC injection at session start (`.cursor/hooks/gxt-session-start.sh`). Other integrations get recipes in `templates/integrations/recipes/` but require manual `source scripts/gxt-runtime-env.sh` + pin script. Adopters on Claude Code, Codex CLI, etc. often run unscoped agents despite documented closed loop. `init-asset-catalog.ts` ships Cursor hooks as managed assets; no session-start equivalents for other IDEs.

## Acceptance

- [ ] Document per-tool bootstrap patterns in `templates/integrations/recipes/` (session init files, env hooks where vendors support them)
- [ ] Where APIs exist, ship template hooks/config (e.g. Claude Code settings, Codex env block) calling `gxt-runtime-env.sh`
- [ ] Optional `gapman session bootstrap --mission …` prints tool-specific setup snippet (`--format cursor|claude|shell`)
- [ ] `gapman doctor` / integration checks flag "detected IDE but no pin/bootstrap configured"
- [ ] Parity test: pinned mission → bootstrap output includes `GXT_TMVC_ROOTS`

## Priority

**Medium** — adoption bucket; reduces onboarding friction for multi-IDE teams.
