## Problem

Long-lived repos or MSN IDs reused after >200 commits can hit false `NO_MSN_COMMITS` / wrong Teacher stamp selection. `git-proof.ts` hardcodes `DEFAULT_MSN_SCAN_DEPTH = 200`; `TeacherMissionProofOptions.scanDepth` exists internally but is **not** wired to CLI, env, or config. Docs repeat "last 200 commits" in `.gitagent/missions/README.md` without override path.

## Acceptance

- [ ] Expose depth via `gapman verify --git-proof-scan-depth <n>` (and/or `GAPMAN_GIT_PROOF_SCAN_DEPTH`)
- [ ] Default remains 200; invalid values fall back with warning
- [ ] `assertTeacherMissionProof` / verify JSON surfaces effective depth on failure
- [ ] Tests in `git-proof.test.ts` for custom depth
- [ ] Update `.gitagent/missions/README.md` and verify error hints in `fix-hints.ts`

## Priority

**Low** — edge case for long history repos; low churn until needed.
