## Problem

`gapman mission validate --file` is documented as JSON Schema validation, but `mission-yaml.ts` uses ad-hoc TypeScript assertions (`assertYamlMissionShape`) that can drift from `.gitagent/teacher/MISSION.schema.yaml`. Invalid missions (unknown fields, malformed `upgrade_payload`) may pass validate yet fail later in verify/upgrade. No Ajv dependency is wired despite schema header saying "Consumed by gapman (Ajv)".

## Acceptance

- [ ] Add Ajv (or equivalent) validation of YAML missions against `MISSION.schema.yaml` in `validateYamlMission` / `runMissionValidate`
- [ ] `--json` errors include schema path + Ajv error messages (instance path, keyword)
- [ ] Reject `additionalProperties` not in schema; accept documented `upgrade_payload` shape
- [ ] Tests: invalid mission fixtures fail validate; valid `example.verify.yaml` and upgrade mission shape pass
- [ ] README / `program-mission.ts` description matches behavior (schema-backed, not "presence checks only")

## Priority

**Major** — tactical maintainability; schema/doc lie creates late failures at verify time.
